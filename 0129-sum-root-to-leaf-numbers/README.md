# 0129-sum-root-to-leaf-numbers

## 📋 Problem Description
You are given the `root` of a binary tree. Each node in the tree contains a single digit from `0` to `9`.
Every path from the root node to a leaf node represents a number. For example, if a path is `1 -> 2 -> 3`, it represents the number `123`.
Your task is to return the total sum of all such numbers formed by every root-to-leaf path in the tree.
A leaf node is defined as a node that has no children (i.e., both its left and right pointers are null).
The test cases are designed such that the final sum will always fit within a 32-bit integer.

## 🔍 Examples
```
Input: root = [1,2,3]
Output: 25
Explanation:
The root-to-leaf path 1->2 represents the number 12.
The root-to-leaf path 1->3 represents the number 13.
Therefore, sum = 12 + 13 = 25.
```

```
Input: root = [4,9,0,5,1]
Output: 1026
Explanation:
The root-to-leaf path 4->9->5 represents the number 495.
The root-to-leaf path 4->9->1 represents the number 491.
The root-to-leaf path 4->0 represents the number 40.
Therefore, sum = 495 + 491 + 40 = 1026.
```

## 📌 Constraints
*   The number of nodes in the tree is in the range `[1, 1000]`.
*   `0 <= Node.val <= 9`
*   The depth of the tree will not exceed `10`.

## 🤔 Understanding the Problem
The problem asks us to interpret each complete path from the root of the binary tree down to any leaf node as a numerical value. For instance, if a path consists of nodes with values `A -> B -> C`, this path forms the number `ABC`. Our goal is to find all such numbers generated by every root-to-leaf path and then sum them up. This requires a systematic way to traverse the tree, build the number as we go deeper, and identify when a path terminates at a leaf.

## 💡 Core Idea
The problem can be efficiently solved by performing a Depth-First Search (DFS) traversal. As we recursively move down the tree, we can construct the current number by appending the value of each visited node. When a leaf node is encountered, the number formed along the path to that leaf is complete and should be added to our running total.

## 🧠 Approach — Depth-First Search (DFS) / Recursion
This problem is a classic tree traversal scenario, perfectly suited for a Depth-First Search (DFS) approach implemented using recursion. DFS allows us to explore each path from the root to a leaf entirely before backtracking. Recursion naturally handles the traversal: for each node, we process it (by incorporating its value into the current number), then recursively call the function for its children. This pattern is ideal because we need to build a "path-dependent" value (the number) as we go deeper into the tree, and recursion's call stack implicitly manages this state for each distinct path.

## 📝 Step-by-Step Algorithm
1.  **Initialize DFS**: Define a recursive helper function, let's call it `dfs(node, current_path_sum)`. The `node` parameter represents the current node being visited, and `current_path_sum` is the number formed by the path from the root down to the *parent* of the current `node`.
2.  **Base Case**: If `node` is `NULL` (meaning we've gone past a leaf or the tree is empty), this path contributes nothing, so return `0`.
3.  **Update Current Path Sum**: When visiting a `node`, update `current_path_sum` by appending the `node->val`. This is done by multiplying `current_path_sum` by 10 and then adding `node->val`. For example, if `current_path_sum` is `12` and `node->val` is `3`, the new `current_path_sum` becomes `12 * 10 + 3 = 123`.
4.  **Leaf Node Check**: After updating `current_path_sum`, check if the current `node` is a leaf node. A node is a leaf if both `node->left` and `node->right` are `nullptr`. If it is a leaf, the `current_path_sum` now represents a complete root-to-leaf number. Return this `current_path_sum`.
5.  **Recursive Step**: If the `node` is not a leaf, it means we need to continue exploring its children. Recursively call `dfs` for its left child (`dfs(node->left, current_path_sum)`) and its right child (`dfs(node->right, current_path_sum)`), passing the `current_path_sum` that includes the current node's value. The total sum contributed by the subtree rooted at the current `node` will be the sum of the results from its left and right children.
6.  **Main Function**: The main `sumNumbers` function will initiate the DFS by calling `dfs(root, 0)`. The initial `current_path_sum` is `0` because no digits have been processed yet. The result of this initial call will be the total sum of all root-to-leaf numbers.

## 💻 Solution

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left), right(right) {}
 * };
 */
class Solution {
public:
    // Helper function to perform Depth-First Search (DFS)
    // root: The current node being visited in the traversal.
    // cursum: The number formed by the path from the tree's root down to the
    //         parent of the current 'root' node. This value is updated as we descend.
    int dfs(TreeNode* root, int cursum) {
        // Base case: If the current node is null, it means we've gone past a leaf
        // or the initial tree itself was empty. This path contributes 0 to the sum.
        if (root == NULL) {
            return 0;
        }

        // Update the current number by appending the current node's value.
        // For example, if cursum is 12 and root->val is 3, the new cursum becomes 123.
        // The original code `cursum=+cursum*10+root->val;` uses a unary plus operator
        // which effectively means `cursum = (cursum * 10) + root->val;`.
        // This correctly calculates the new path sum.
        cursum = cursum * 10 + root->val;


        // Check if the current node is a leaf node.
        // A leaf node is defined as a node with no left child and no right child.
        if (root->left == nullptr && root->right == nullptr) {
            // If it's a leaf, the 'cursum' now represents a complete root-to-leaf number.
            // Return this number as its contribution to the total sum.
            return cursum;
        }

        // If it's not a leaf node, we need to continue the DFS traversal.
        // Recursively call dfs for the left child and the right child,
        // passing the 'cursum' that now includes the current node's value.
        // The total sum from this node's subtree is the sum of numbers
        // found in its left subtree and its right subtree.
        return dfs(root->left, cursum) + dfs(root->right, cursum);
    }

    // Main function to calculate the sum of all root-to-leaf numbers.
    int sumNumbers(TreeNode* root) {
        // Start the DFS traversal from the root node.
        // Initialize the current sum for the path as 0, as we haven't processed any digits yet.
        return dfs(root, 0);
    }
};

```

## ⏱️ Complexity Analysis

| | Complexity | Reason |
|---|---|---|
| **Time** | O(N) | Each node in the binary tree is visited exactly once during the Depth-First Search traversal. N is the total number of nodes in the tree. |
| **Space** | O(H) | The space complexity is determined by the maximum depth of the recursion stack, which corresponds to the height (H) of the tree. In the worst case (a skewed tree), H can be N. In the best case (a perfectly balanced tree), H is log N. Given the constraint that the tree depth will not exceed 10, the space complexity is effectively O(10), which is constant. |
