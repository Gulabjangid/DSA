# 0232-implement-queue-using-stacks

## 📋 Problem Description
Implement a First-In, First-Out (FIFO) queue using only two Last-In, First-Out (LIFO) stacks. The implemented queue must support the standard queue operations: `push`, `pop`, `peek`, and `empty`.

You need to implement the `MyQueue` class with the following methods:
*   `void push(int x)`: Adds element `x` to the back of the queue.
*   `int pop()`: Removes and returns the element from the front of the queue.
*   `int peek()`: Returns the element at the front of the queue without removing it.
*   `boolean empty()`: Returns `true` if the queue is empty, `false` otherwise.

You are restricted to using only standard stack operations: `push to top`, `peek/pop from top`, `size`, and `is empty`.

## 🔍 Examples
```
Input:
["MyQueue", "push", "push", "peek", "pop", "empty"]
[[], [1], [2], [], [], []]
Output:
[null, null, null, 1, 1, false]
Explanation:
MyQueue myQueue = new MyQueue();
myQueue.push(1); // queue is: [1]
myQueue.push(2); // queue is: [1, 2] (leftmost is front of the queue)
myQueue.peek();  // returns 1
myQueue.pop();   // returns 1, queue is [2]
myQueue.empty(); // returns false
```

```
Input:
["MyQueue", "push", "empty", "push", "pop", "peek"]
[[], [5], [], [10], [], []]
Output:
[null, null, false, null, 5, 10]
Explanation:
MyQueue myQueue = new MyQueue();
myQueue.push(5);   // queue is: [5]
myQueue.empty();   // returns false
myQueue.push(10);  // queue is: [5, 10]
myQueue.pop();     // returns 5, queue is [10]
myQueue.peek();    // returns 10
```

## 📌 Constraints
*   `1 <= x <= 9`
*   At most `100` calls will be made to `push`, `pop`, `peek`, and `empty`.
*   All calls to `pop` and `peek` are guaranteed to be valid (i.e., the queue will not be empty when these operations are called).

## 🤔 Understanding the Problem
The core challenge is to simulate a First-In, First-Out (FIFO) data structure (a queue) using only Last-In, First-Out (LIFO) data structures (stacks). This means that the element added first must be the first one removed, but stacks naturally remove the element added last. We need a strategy to reverse the order of elements when necessary to maintain the FIFO property.

## 💡 Core Idea
To make the "first-in" element accessible for `pop` and `peek` operations, we can use a second auxiliary stack to temporarily reverse the order of elements. By moving all elements from the primary stack to the auxiliary stack, the original "bottom" element (which is the "first-in") becomes the "top" of the auxiliary stack.

## 🧠 Approach — Stack Manipulation
This problem is a classic example of **Stack Manipulation** for data structure design. The pattern fits because we are explicitly asked to build one data structure (queue) using another (stack) with different fundamental access patterns. By using two stacks, we can effectively "flip" the order of elements. One stack (`s1`) will be used to store elements in a way that its top represents the front of the queue, while the second stack (`s2`) acts as a temporary buffer to facilitate this reordering during `push` operations.

## 📝 Step-by-Step Algorithm

The solution uses two stacks: `s1` (main stack) and `s2` (auxiliary stack). The goal is to always keep the front of the queue at the top of `s1`.

1.  **`MyQueue()` Constructor**:
    *   Initialize the two internal stacks, `s1` and `s2`.

2.  **`push(int x)`**: (Adds `x` to the back of the queue)
    *   To ensure `x` ends up at the "bottom" of `s1` (which represents the back of the queue), we need to move all existing elements out of the way.
    *   While `s1` is not empty, pop elements from `s1` and push them onto `s2`. This effectively reverses the order of elements from `s1` into `s2`.
    *   Now, `s1` is empty. Push the new element `x` onto `s1`. This `x` is now at the "bottom" of what will become the re-ordered `s1`.
    *   While `s2` is not empty, pop elements from `s2` and push them back onto `s1`. This restores the original elements to `s1`, but now with `x` inserted at the very bottom, maintaining the FIFO order where the front of the queue is at `s1.top()`.

3.  **`pop()`**: (Removes and returns the front element)
    *   Since `s1` is always maintained such that its top element is the front of the queue, simply return `s1.top()` and then `s1.pop()`.

4.  **`peek()`**: (Returns the front element without removing it)
    *   Similarly, the front of the queue is always at `s1.top()`. Just return `s1.top()`.

5.  **`empty()`**: (Checks if the queue is empty)
    *   The queue is empty if and only if `s1` is empty. Return `s1.empty()`.

## 💻 Solution

```cpp
#include <stack> // Required for std::stack

class MyQueue {
public:
    // s1 will store elements such that its top is the front of the queue.
    // s2 will be used as an auxiliary stack for reordering elements during push.
    std::stack<int> s1;
    std::stack<int> s2;

    MyQueue() {
        // Constructor: Stacks are automatically initialized as empty.
    }
    
    // Pushes element x to the back of the queue.
    void push(int x) {
        // Step 1: Move all elements from s1 to s2.
        // This reverses their order, so the element that was at the bottom of s1
        // (which was the front of the queue) is now at the top of s2.
        while (!s1.empty()) {
            s2.push(s1.top()); // Move element from s1's top to s2's top
            s1.pop();          // Remove element from s1
        }
        
        // Step 2: Push the new element x onto s1.
        // Since s1 is now empty, x becomes the new "bottom" element of s1,
        // effectively placing it at the back of our conceptual queue.
        s1.push(x);
        
        // Step 3: Move all elements back from s2 to s1.
        // This restores the original elements to s1, but now with x
        // inserted at the very bottom. The order is preserved such that
        // the front of the queue remains at s1.top().
        while (!s2.empty()) {
            s1.push(s2.top()); // Move element from s2's top back to s1's top
            s2.pop();          // Remove element from s2
        }
    }
    
    // Removes the element from the front of the queue and returns it.
    int pop() {
        // The front of the queue is always at the top of s1 in this implementation.
        int ans = s1.top(); // Get the front element
        s1.pop();           // Remove the front element
        return ans;         // Return the removed element
    }
    
    // Returns the element at the front of the queue.
    int peek() {
        // The front of the queue is always at the top of s1.
        return s1.top(); // Return the front element without removing it
    }
    
    // Returns true if the queue is empty, false otherwise.
    bool empty() {
        // The queue is empty if and only if the main stack s1 is empty.
        // The expression `true?s1.empty():false` is equivalent to `s1.empty()`.
        return s1.empty(); 
    }
};

/**
 * Your MyQueue object will be instantiated and called as such:
 * MyQueue* obj = new MyQueue();
 * obj->push(x);
 * int param_2 = obj->pop();
 * int param_3 = obj->peek();
 * bool param_4 = obj->empty();
 */
```
