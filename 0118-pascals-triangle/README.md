# 0118-pascals-triangle

## 📋 Problem Description
Given an integer `numRows`, the task is to generate the first `numRows` rows of Pascal's triangle. In Pascal's triangle, each number is derived by summing the two numbers directly above it. The edges of the triangle are always 1s.

The function `generate` receives one integer, `numRows`, as input. It must return a `vector<vector<int>>` (or list of lists of integers in other languages), where each inner vector represents a row of Pascal's triangle.

## 🔍 Examples
```
Input: numRows = 5
Output: [[1],[1,1],[1,2,1],[1,3,3,1],[1,4,6,4,1]]
Explanation:
Row 0: [1]
Row 1: [1,1]
Row 2: [1,2,1] (1+1=2)
Row 3: [1,3,3,1] (1+2=3, 2+1=3)
Row 4: [1,4,6,4,1] (1+3=4, 3+3=6, 3+1=4)
```

```
Input: numRows = 1
Output: [[1]]
Explanation: Only the first row is requested.
```

## 📌 Constraints
*   `1 <= numRows <= 30`

## 🤔 Understanding the Problem
The problem asks us to construct Pascal's triangle up to a specified number of rows. The core rule is that every number in the triangle (except for the ones at the very edges) is the sum of the two numbers directly above it. The numbers at the beginning and end of each row are always 1. This means we need to build the triangle row by row, using the values from the previously generated row to compute the current one. The problem is non-trivial because each row depends on the one before it, requiring careful construction.

## 💡 Core Idea
The fundamental property of Pascal's triangle is that each element (excluding the boundary 1s) is the sum of the two elements directly above it in the previous row. This allows us to build the triangle iteratively, row by row, by always referencing the immediately preceding row.

## 🧠 Approach — Iterative Construction with Dynamic Programming Principles
This problem is perfectly suited for an iterative construction approach, which inherently utilizes dynamic programming principles. We build the solution for `numRows` by leveraging the solutions for `numRows-1`, `numRows-2`, and so on. Each new row is generated by directly referencing the elements of the previously completed row. This method efficiently stores and reuses intermediate results (the previous rows), avoiding redundant calculations and directly applying the recursive definition of Pascal's triangle.

## 📝 Step-by-Step Algorithm
1.  Initialize an empty list of lists, `result`, which will store all the rows of Pascal's triangle.
2.  Start a loop that iterates `numRows` times. Let `i` be the current row index, starting from `0`.
    a.  For each `i`, create a new list of integers, `currentRow`, of size `i + 1`.
    b.  Set the first element (`currentRow[0]`) and the last element (`currentRow[i]`) of `currentRow` to 1. These are always 1 in Pascal's triangle.
    c.  If `i` is greater than 1 (meaning we are processing a row that has elements *between* the first and last 1s), iterate with an inner loop from `j = 1` up to `i - 1`.
        i.  For each `j`, calculate `currentRow[j]` by summing the elements from the *previous* row (`result[i - 1]`). Specifically, `currentRow[j]` will be `result[i - 1][j - 1] + result[i - 1][j]`.
    d.  After `currentRow` is fully constructed, add it to the `result` list.
3.  Once the outer loop completes, `result` will contain all `numRows` of Pascal's triangle. Return `result`.

## 💻 Solution
```cpp
class Solution {
public:
    vector<vector<int>> generate(int numRows) {
        // This vector of vectors will store all the rows of Pascal's triangle.
        // Each inner vector represents a single row.
        vector<vector<int>> result;

        // Iterate 'numRows' times to generate each row of the triangle.
        // 'i' represents the current row index (0-indexed).
        for (int i = 0; i < numRows; i++) {
            // Create a new vector for the current row.
            // A row 'i' has 'i + 1' elements.
            // Initialize all elements to 1. This conveniently sets the first and last elements
            // of any row to 1, as they are always 1 in Pascal's triangle.
            vector<int> row(i + 1, 1);

            // Calculate the inner elements of the current row.
            // This loop runs from j=1 up to i-1.
            // It effectively skips the first and last elements (which are already 1).
            // This condition `j < i` ensures it runs only for rows with more than 2 elements (i.e., i > 1).
            // For i=0 (row [1]) and i=1 (row [1,1]), this loop won't execute, which is correct.
            for (int j = 1; j < i; j++) {
                // Each element in Pascal's triangle is the sum of the two elements directly above it.
                // 'result[i - 1]' refers to the previous row that has already been computed and stored.
                // 'result[i - 1][j - 1]' is the element directly above and to the left in the previous row.
                // 'result[i - 1][j]' is the element directly above and to the right in the previous row.
                row[j] = result[i - 1][j - 1] + result[i - 1][j];
            }

            // Add the newly constructed row to our overall result.
            result.push_back(row);
        }

        // Return the complete Pascal's triangle.
        return result;
    }
};
```

## ⏱️ Complexity Analysis
| | Complexity | Reason |
|---|---|---|
| **Time** | O(numRows^2) | We iterate `numRows` times to generate each row. For each row `i`, we create a vector of size `i+1` and then iterate `i-1` times to fill its inner elements. The total number of operations is proportional to the sum `1 + 2 + ... + numRows`, which is `numRows * (numRows + 1) / 2`, simplifying to O(numRows^2). |
| **Space** | O(numRows^2) | We store `numRows` rows. The `i`-th row has `i+1` elements. The total number of elements stored across all rows is `1 + 2 + ... + numRows`, which is `numRows * (numRows + 1) / 2`, simplifying to O(numRows^2). This is the space required to store the output. |
